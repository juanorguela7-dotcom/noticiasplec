🔍 CÓMO PROBAR EN TU NAVEGADOR
═══════════════════════════════════════════════════════════════════════════════

OPCIÓN 1: Device Emulation en Chrome/Edge
──────────────────────────────────────────
1. Abre la página en Chrome o Edge
2. Presiona F12 (o Ctrl+Shift+I)
3. Presiona Ctrl+Shift+M para "Device Mode"
4. En el dropdown superior, selecciona:
   • iPhone SE (375px)
   • iPhone 12 Pro (390px)
   • Galaxy S21 (360px)
   • iPad Air (768px)
   • Responsive (redimensiona manualmente)

OPCIÓN 2: Redimensionamiento Manual
────────────────────────────────────
1. Abre DevTools (F12)
2. Click en el icono de "Responsive" 
3. En Device Mode, selecciona "Responsive"
4. Arrastra el borde derecho para cambiar ancho
5. Observa cómo cambian los breakpoints

OPCIÓN 3: Firefox Developer
─────────────────────────
1. Presiona F12
2. Click en el icono de Responsive Design
3. Selecciona dispositivos predefinidos
4. O usa "Edit List" para crear custom

OPCIÓN 4: Safari Developer Tools (iOS)
──────────────────────────────────
1. En iPhone/iPad: Configuración → Safari
2. Scroll down → Opciones Avanzadas
3. Web Inspector → Activar
4. En la Mac: Presiona Cmd+Option+U

✨ PUNTOS CLAVE DEL RESPONSIVE DESIGN
═══════════════════════════════════════════════════════════════════════════════

1. VIEWPORT META TAG ✓
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   ▶ Asegura que el navegador respete el tamaño del dispositivo
   ▶ Impide zoom automático innecesario
   ▶ Permite media queries

2. BREAKPOINTS GRANULARES ✓
   • 320px - Móvil muy pequeño
   • 360px - Móvil compacto
   • 420px - Móvil pequeño
   • 480px - Móvil mediano
   • 650px - Móvil grande
   • 768px - Tablet pequeña
   • 1024px - Tablet estándar
   • 1199px - Tablet grande
   • 1200px+ - Desktop

3. TOUCH TARGETS ✓
   • Botones: mínimo 44x44px (WCAG recomendado)
   • Inputs: mínimo 32px altura
   • Espaciado: 8-30px entre elementos
   • Tapping sin errores garantizado

4. FLEXIBLE LAYOUTS ✓
   • Grillas: 1-4 columnas dinámicas
   • Flexbox: flex-wrap automático
   • Padding/Margin: proporcional al tamaño
   • Max-widths: adaptan al contenedor

5. IMAGES RESPONSIVAS ✓
   • object-fit: cover (sin distorsión)
   • Heights: ajustables por breakpoint
   • Widths: 100% del contenedor
   • Carga eficiente

🐛 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA: Scroll horizontal en móvil
SOLUCIÓN:
• Verifica: overflow-x en body
• Revisa: width de elementos
• Usa: box-sizing: border-box
• Checkea: max-width: 100%

PROBLEMA: Texto muy pequeño en móvil
SOLUCIÓN:
• Aumenta font-size en media query
• Asegura mínimo 16px en inputs
• Aumenta line-height
• Usa rem en lugar de px

PROBLEMA: Botones difíciles de tocar
SOLUCIÓN:
• Aumenta padding: min 44x44px
• Aumenta min-height
• Aumenta gap entre botones
• Prueba con dedo real

PROBLEMA: Imágenes pixeladas
SOLUCIÓN:
• Sube imágenes HD
• Usa srcset si posible
• Verifica object-fit: cover
• Aumenta height de contenedor

PROBLEMA: Navegación congestionada
SOLUCIÓN:
• Usa hamburger menu en móvil
• Apila items verticalmente
• Reduce font-size
• Usa flex-wrap: wrap

🚀 MEJORES PRÁCTICAS
═══════════════════════════════════════════════════════════════════════════════

✓ Mobile-First Approach
  • Diseña primero para móvil (320px)
  • Luego expande para tablets/desktop
  • Media queries: max-width decreciente

✓ Flexible Units
  • Usa %, em, rem en lugar de px
  • Usa max-width: 100%
  • Usa min-height/width cuando apropiado

✓ Touch-Friendly
  • Mínimo 44x44px para targets
  • 8px gap entre elementos
  • Feedback visual en tap

✓ Performance
  • Optimiza imágenes
  • Minimiza CSS/JS
  • Lazy load donde posible
  • Compress assets

✓ Accessibility
  • Contraste suficiente (WCAG AA)
  • Font legible (mínimo 16px inputs)
  • Focus states visibles
  • Semántica HTML correcta
